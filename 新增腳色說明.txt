新增角色說明
==============

這份說明的目標是讓新角色可以用「相對路徑」接進目前這份大專案，而不是再綁死某台機器的 `C:\...` 位置。

原則
----

新增一個角色時，請先決定角色 ID。  
建議只使用英文小寫、數字、底線，例如：

- `kuro`
- `yumi`
- `mao_pro`
- `shizuku`

之後會盡量讓這個 ID 同時出現在：

- `Open-LLM-VTuber/characters/<角色ID>.yaml`
- `live2d_model_name`
- `voices/<角色ID>/`
- `gpt_sovits/GPT_SoVITS/configs/tts_infer_<角色ID>.yaml`

必要檔案
--------

至少要準備這幾塊：

1. 角色 YAML  
   `Open-LLM-VTuber/characters/<角色ID>.yaml`

2. persona prompt  
   `Open-LLM-VTuber/prompts/persona/<角色ID>.txt`

3. Live2D 模型資料夾  
   `Open-LLM-VTuber/live2d-models/<角色ID>/`

4. Live2D 對應資料  
   `Open-LLM-VTuber/model_dict.json`

5. 參考音檔  
   `voices/<角色ID>/ref.wav`
   或  
   `voices/<角色ID>/ref6s.wav`

6. GPT-SoVITS infer 設定  
   `gpt_sovits/GPT_SoVITS/configs/tts_infer_<角色ID>.yaml`

7. 權重檔  
   `gpt_sovits/GPT_weights_v2Pro/...`
   `gpt_sovits/SoVITS_weights_v2Pro/...`

如果某個角色目前還沒有完整資料，可以先建立 placeholder，之後再慢慢補。

步驟 1：建立角色 YAML
--------------------

請先複製一份現有角色 YAML，例如 `kuro.yaml`，再改成新的角色 ID：

- `Open-LLM-VTuber/characters/<角色ID>.yaml`

建議至少確認這些欄位：

- `conf_name`
- `conf_uid`
- `live2d_model_name`
- `character_name`
- `human_name`
- `persona_prompt_path`
- `agent_config`
- `tts_config`

路徑類欄位要使用相對寫法，例如：

```yaml
character_config:
  conf_name: new_character
  conf_uid: new_character_01
  live2d_model_name: new_character
  character_name: new_character
  human_name: new_character
  persona_prompt_path: prompts/persona/new_character.txt

tts_config:
  tts_model: gpt_sovits_tts
  gpt_sovits_tts:
    api_url: http://127.0.0.1:9881/tts
    text_lang: ja
    ref_audio_path: voices/new_character/ref.wav
    prompt_lang: ja
    prompt_text: ""
```

注意：

- `conf_uid` 最好保持唯一
- `ref_audio_path` 現在應該寫成 `voices/...`，不要再寫成 `C:/...`

步驟 2：建立 persona prompt
--------------------------

在這裡建立角色人格：

- `Open-LLM-VTuber/prompts/persona/<角色ID>.txt`

這份只放角色個性、語氣、稱呼、互動習慣。  
不要把工具規則、專案規則、系統輸出格式混進去。

步驟 3：接 Live2D
----------------

模型資料夾：

- `Open-LLM-VTuber/live2d-models/<角色ID>/`

同時要在這裡補上角色對應：

- `Open-LLM-VTuber/model_dict.json`

至少要確認：

- `name`
- `description`
- `url`
- `kScale`
- `idleMotionGroupName`
- `emotionMap`

範例：

```json
{
  "name": "new_character",
  "description": "new_character Live2D",
  "url": "/live2d-models/new_character/runtime/new_character.model3.json",
  "kScale": 0.45,
  "idleMotionGroupName": "Idle",
  "defaultEmotion": 0,
  "emotionMap": {
    "neutral": 0,
    "joy": 1,
    "smirk": 1,
    "surprise": 2,
    "anger": 3,
    "sadness": 4,
    "fear": 5,
    "disgust": 6
  }
}
```

步驟 4：接 TTS 參考音
---------------------

角色音檔目錄：

- `voices/<角色ID>/`

常用檔名：

- `ref.wav`
- `ref6s.wav`

這些音檔通常不提交到 git，但角色 YAML 會透過相對路徑引用它們。

步驟 5：接 GPT-SoVITS infer 設定
--------------------------------

位置：

- `gpt_sovits/GPT_SoVITS/configs/tts_infer_<角色ID>.yaml`

如果新角色還沒有完整權重，可以先放 placeholder：

```yaml
custom:
  bert_base_path: GPT_SoVITS/pretrained_models/chinese-roberta-wwm-ext-large
  cnhuhbert_base_path: GPT_SoVITS/pretrained_models/chinese-hubert-base
  device: cuda
  is_half: true
  t2s_weights_path: ""
  version: v2Pro
  vits_weights_path: ""
```

之後補齊：

- `t2s_weights_path`
- `vits_weights_path`

步驟 6：沒有資料時的 placeholder 規則
-----------------------------------

如果目前只有角色框架，還沒有完整資源，建議先補這些最小佔位：

- `Open-LLM-VTuber/characters/<角色ID>.yaml`
- `Open-LLM-VTuber/prompts/persona/<角色ID>.txt`
- `gpt_sovits/GPT_SoVITS/configs/tts_infer_<角色ID>.yaml`

音檔或權重尚未完成時：

- `ref_audio_path` 先指向未來預計位置
- `prompt_text` 可先留空
- 權重路徑可先留空字串

步驟 7：用 launcher 驗證
----------------------

從 repo 根目錄啟動：

```powershell
cd <repo-root>
.\envs\kuro-llm310\python.exe .\launcher.py
```

確認這些項目：

- 角色有出現在 launcher 列表
- `conf_uid` 沒和別人撞名
- `live2d_model_name` 在 `model_dict.json` 有對應
- Live2D `.model3.json` 路徑正確
- TTS 設定可讀
- 參考音檔存在且不是空檔
- `tts_infer_<角色ID>.yaml` 路徑正確
- GPT / SoVITS 權重路徑正確，或確定目前只是 placeholder

Git 注意事項
------------

建議提交：

- 角色 YAML
- persona prompt
- `model_dict.json`
- `tts_infer_<角色ID>.yaml`
- Live2D 結構檔

不要提交：

- `.env`
- API key
- `launcher_logs/`
- `Open-LLM-VTuber/conf.launcher_runtime.yaml`
- `voices/**/*.wav`
- `*.ckpt`
- `*.pth`
- `gpt_sovits/GPT_weights*/`
- `gpt_sovits/SoVITS_weights*/`
- `gpt_sovits/GPT_SoVITS/pretrained_models/`

新增角色檢查清單
----------------

- [ ] `Open-LLM-VTuber/characters/<角色ID>.yaml` 已建立
- [ ] `conf_name` 已改成角色名稱
- [ ] `conf_uid` 唯一
- [ ] `live2d_model_name` 正確
- [ ] `persona_prompt_path` 已改成對應角色檔
- [ ] `Open-LLM-VTuber/prompts/persona/<角色ID>.txt` 已建立
- [ ] `Open-LLM-VTuber/model_dict.json` 已補上角色
- [ ] Live2D `.model3.json` 路徑正確
- [ ] `voices/<角色ID>/` 已建立
- [ ] `ref_audio_path` 使用相對路徑
- [ ] `gpt_sovits/GPT_SoVITS/configs/tts_infer_<角色ID>.yaml` 已建立
- [ ] 權重路徑已補齊，或確認目前為 placeholder
- [ ] launcher 能正常讀到角色
